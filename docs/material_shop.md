# 教材ストア（販売サイト）の使い方

自作教材（PDF・Excelなど）を販売するための自前ストアサイトです。
GitHub Pages で無料公開でき、Google検索にも載せられるようにSEO対応済みです。

- コード: `web/shop/`
- 公開URL（Pages有効化後）: `https://blitzendegen915-beep.github.io/edusup/`

## 仕組み

```
products.json（商品データ）＋ site.json（サイト設定）
        │
        ▼  node web/shop/build.js
web/shop/dist/（トップページ・商品ページ・sitemap.xml・robots.txt）
        │
        ▼  GitHub Actions（main に push すると自動実行）
GitHub Pages で公開
```

- サーバーもデータベースも不要。維持費は0円です。
- 決済は Stripe の「支払いリンク（Payment Links）」を使うので、決済処理を自分で作る必要はありません。

## 初回セットアップ（1回だけ）

1. このブランチを main にマージする（またはファイルを main に置く）。
2. GitHubのリポジトリページで **Settings → Pages** を開く。
3. **Source** を **GitHub Actions** に変更する。
4. **Actions** タブで「Deploy shop to GitHub Pages」が成功するのを待つ。
5. `https://blitzendegen915-beep.github.io/edusup/` を開いて表示を確認する。

### site.json を自分用に書き換える

`web/shop/site.json` の以下を必ず書き換えてください（今はダミーです）。

- `authorName` / `contactEmail`: あなたの名前（屋号）と連絡先
- `tokushoho`: 特定商取引法に基づく表記の内容
  - 有料販売する場合は法律上必要な表記です。個人の場合、住所・電話番号は「請求があった場合に遅滞なく開示します」という書き方が広く使われています。

## 想定しているお客様（ペルソナ）

このストアは3種類のお客様を想定しています。商品登録時に「対象」を必ず設定してください。トップページで対象別に絞り込めるほか、紹介文の書き分けの軸になります。

| 対象 | 求めているもの | 商品づくり・紹介文のコツ |
| --- | --- | --- |
| 大人の英語学習者 | 学び直し・独学用の分かりやすい教材 | 「社会人」「学び直し」「スマホで読める」など生活に寄せた言葉を使う |
| 高校生 | 定期テスト・受験対策の演習素材 | 「定期テスト」「共通テスト」「〇分でできる」など具体的な場面を書く |
| 教員 | そのまま授業・校務に使える時短ツール | 「印刷してすぐ使える」「マクロ不要」「解答つき」など手間が減ることを書く |

## 商品を追加する

### 方法1: 管理画面から（おすすめ）

公開サイトの `/admin/`（例: `https://blitzendegen915-beep.github.io/edusup/admin/`）を開くと、ブラウザ上のフォームで商品の追加・編集・削除・並べ替え・無料配布ファイルのアップロードができます。

初回だけGitHubのアクセストークンが必要です。

1. GitHub → 右上アイコン → **Settings** → 左下 **Developer settings** → **Fine-grained personal access tokens** → **Generate new token**
2. Repository access: **Only select repositories** → このリポジトリだけを選ぶ
3. Permissions → Repository permissions → **Contents** を **Read and write** にする
4. 生成されたトークン（`github_pat_...`）を管理画面に貼り付けて「接続」

トークンは自分のブラウザ（localStorage）にだけ保存されます。共用PCでは使い終わったら「トークンを削除」を押してください。管理画面は検索エンジンにインデックスされない設定（noindex + robots.txt）にしてありますが、トークンがなければ誰が開いても保存はできません。

編集の流れ: 商品を選ぶ → フォームを直す → **「この内容を一覧に反映」** → 最後に **「保存して公開」**。1〜2分で自動的にサイトへ反映されます。

### サンプル画像を載せる（購入率アップの最重要ポイント）

教材の1ページ目などを画像にして商品に載せられます。**中身が1枚見えるだけで購入率が大きく変わる**ので、必ず設定するのがおすすめです。

- 管理画面の「サンプル画像」欄の下にあるアップロードボタンから画像（PNG/JPEG/WebP、20MBまで）を上げると、自動でファイル名が入ります
- 複数枚のときはカンマ区切り。**1枚目がトップページのサムネイルとSNSシェア時の画像**（OGP）になります
- 画像の作り方: 教材PDFの1ページ目をスクリーンショットして「SAMPLE」等の透かしを入れるだけで十分です。推奨サイズは横1200px程度
- 解答ページは載せないよう注意してください

### 方法2: products.json を直接編集する

`web/shop/products.json` に1ブロック追記するだけです。

```json
{
  "id": "my-new-worksheet",
  "title": "商品名（検索されたいキーワードを入れる）",
  "category": "ワークシート",
  "audience": ["高校生", "教員"],
  "price": 300,
  "fileType": "PDF",
  "summary": "検索結果にそのまま表示される紹介文。80〜120文字程度がおすすめ。",
  "details": ["内容の箇条書き1", "内容の箇条書き2"],
  "tags": ["高校英語", "文法"],
  "paymentLink": "",
  "downloadFile": "",
  "updated": "2026-07-03"
}
```

- `id` は半角英小文字・数字・ハイフンのみ。商品ページのURLになります。
- `price` を `0` にして `downloadFile` にファイル名を書くと無料配布になります（ファイルは `web/shop/files/` に置く）。
- main に push すれば自動で再デプロイされます。

## 有料販売（Stripe決済）をつなぐ

1. [Stripe](https://stripe.com/jp) でアカウントを作る（個人でも可・初期費用なし・手数料は売上の3.6%）。
2. ダッシュボードで **商品カタログ → 商品を追加**（価格を設定）。
3. **支払いリンク（Payment Links）を作成** し、「支払い後 → ファイルをダウンロードさせる」設定として、確認ページまたは確認メールにダウンロードURLを入れる。
   - シンプルにやるなら、Stripeの「支払い後にカスタムメッセージを表示」にダウンロードリンクを書くのが最初は楽です。
4. できたリンク（`https://buy.stripe.com/...`）を管理画面の「Stripe支払いリンク」欄（または `products.json` の `paymentLink`）に貼る。
5. 保存すると商品ページの「販売準備中」が「購入する」ボタンに変わります。

### クレジットカード以外の決済（PayPay・コンビニ払い）を有効にする

Stripeはクレジットカードに加えて **PayPay・コンビニ決済** などの日本向け決済に対応しています。コードの変更は不要で、Stripeダッシュボード側で有効にするだけで、同じ支払いリンクの決済画面に選択肢として自動で表示されます。

1. Stripeダッシュボード → **設定（歯車）→ 決済手段（Payment methods）** を開く
2. **PayPay** の「有効にする」を申請する（Stripe経由の利用申請と審査があります。数日かかることがあります）
3. **コンビニ決済（Konbini）** も同様に有効化を申請する
4. 審査が通ると、既存の支払いリンクにも自動で反映されます

補足:

- PayPay・コンビニ決済は最低金額の制約（目安として100円以上）や、カードと異なる手数料率が設定されています。ダッシュボードの表示を確認してください。
- コンビニ決済は入金確認に時間がかかる（お客様が店頭で支払うまで確定しない）ため、「決済完了メールにダウンロードリンクを入れる」方式との相性が良いです。
- サイト上の「対応決済」表示は `web/shop/site.json` の `paymentMethods` で変えられます（Stripe側で有効化できた手段に合わせて書き換えてください）。

※ 販売ファイル自体はこのリポジトリに置かないでください（公開リポジトリだと誰でも見えます）。Stripe経由で渡すか、Googleドライブの限定リンク等を使ってください。

## Google検索に載せる

サイト側の対策（タイトル・説明文・構造化データ・sitemap.xml・robots.txt）は生成時に自動で入ります。あとは Google に場所を教えるだけです。

1. [Google Search Console](https://search.google.com/search-console) を開き、「URLプレフィックス」でサイトURLを登録する。
2. 所有権の確認は「HTMLタグ」方式が楽です。表示されたメタタグを `web/shop/build.js` の `<head>` 部分（`pageShell` 関数内）に追記して push。
3. 確認が済んだら、**サイトマップ** メニューで `sitemap.xml` を送信する。
4. 数日〜数週間でインデックスされます。**URL検査** で個別ページの登録リクエストも出せます。

検索されるためのコツ:

- 商品タイトルに「高校英語 仮定法 ワークシート」のように、先生が実際に検索する語を入れる
- `summary` は検索結果の説明文になるので、内容と対象者が分かる文にする
- 無料教材を1つ置いておくと、そこから他の商品ページへ回遊されやすくなります

## 固定ページ（制作者紹介・プライバシーポリシー）

- **制作者紹介（/about.html）**: `site.json` の `about` を書き換えると内容が変わります。Googleは「誰が作ったか」（E-E-A-T）を評価するので、経歴や教材づくりへのこだわりを自分の言葉で書くとSEOにも効きます。
- **プライバシーポリシー（/privacy.html）**: 自動生成されます。お問い合わせ・Stripe決済・Google Analyticsの扱いをカバー済みなので、基本はそのままで大丈夫です。

## アクセス解析（Google Analytics）を入れる

どの商品ページが見られているかを知るために、公開したら早めに入れるのがおすすめです。

1. [Google Analytics](https://analytics.google.com/) でアカウントとプロパティ（GA4）を作成する
2. 「データストリーム」→「ウェブ」でサイトURLを登録し、**測定ID（G-XXXXXXXXXX）** をコピーする
3. `web/shop/site.json` の `googleAnalyticsId` に貼り付けて push する（空欄のままなら解析タグは入りません）

## 無料コンテンツ（英単語ゲーム）との連携

トップページ下部に、無料で遊べる「英単語バトルレート」（vocab-battle）への誘導セクションが表示されます。ゲーム側のフッターからもストアへリンクしているので、無料ユーザーが商品ページに回遊する導線になっています。表示内容とリンク先は `site.json` の `freeContent` で変更できます（`url` を空にすると非表示）。

## ローカルで確認する

```
node web/shop/build.js
```

を実行すると `web/shop/dist/` にサイトが生成されます。`dist/index.html` をブラウザで開けばそのまま確認できます。

## テスト方法

1. `node web/shop/build.js` がエラーなく完了する
2. `dist/index.html` で商品カードとカテゴリ絞り込みが動く
3. 各商品ページで価格・購入ボタン・パンくずが正しい
4. `dist/sitemap.xml` に全ページのURLが入っている

## 注意点

- 教材に教科書本文・入試問題をそのまま載せると著作権に触れる場合があります。販売物は自作の問題・解説にしてください。
- 有料販売を始める前に `tokushoho`（特定商取引法に基づく表記）を必ず実情に合わせて書き換えてください。
- `web/shop/dist/` は自動生成物なのでコミット不要です（`.gitignore` 済み）。
