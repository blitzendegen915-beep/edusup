# rugby-check: 配布前ファクトチェック

駒澤大学高等学校ラグビー部で保護者・外部向けに配布する文書のファクトチェックを行う。この部では過去に「7月1日(火)と記載したが実際は(水)だった」「ショルダーガード10名分のサイズを誤記した」という事故を配布後に起こしている。同種の事故を配布前に防ぐことが目的であり、目視確認だけで済ませない。

## 検証項目

文書内の該当項目ごとに、以下をすべて確認する。存在しない項目（表がない文書など）はスキップしてよいが、その旨を報告する。

### 1. 曜日

文書中に登場する**すべての日付**について、記載されている曜日をPythonの `datetime` で検算する。記憶や目視で「合っているはず」と判断しない。

```python
import datetime
datetime.date(2026, 7, 1).weekday()  # 0=月, 1=火, ... 6=日
```

複数の日付がある文書（合宿の日程表など）は1件ずつ、日付と曜日表記のペアをすべて検算する。

### 2. 金額

- 合計金額・人数割の金額はコードで再計算し、文書記載の金額と一致するか確認する。
- 単価×数量の計算が正しいか。
- 割り算を使っている金額（グラウンド使用料÷人数、交通費÷人数など）は、**分母の人数が根拠資料と一致しているか**を確認する（例: 部員42名で割るべきところを合宿参加46名で割っていないか、選手38名のみで割るべきところを混同していないか）。

### 3. サイズ・個数

注文サイズ・数量は、Googleフォーム回答など一次データと**1件ずつ**突合する。すでに整列・転記済みの表を鵜呑みにせず、別の項目（例: ヘッドギアのサイズとショルダーガードのサイズ）を混同していないか個別に確認する。

### 4. 氏名・クラス・出席番号

生徒個人別に作成した文書は、`scripts/create_handouts.py` の `STUDENTS` リストと氏名・クラス・出席番号を照合する。同姓・似た氏名の生徒を取り違えていないか確認する。

### 5. 口座・連絡先

`docs/rugby/基礎データ.md` の口座情報と照合する。特に **三菱UFJ銀行（家庭からの振込用）と三井住友銀行（予算入金用）の取り違え**に注意する。文書の性質（保護者が振り込む文書か、部内予算処理の文書か）に応じて正しい口座が指定されているか確認する。

### 6. 人数の整合

文書内で使われている人数（例: 部員42名／合宿参加46名／選手38名）が、その文脈で正しい人数かを確認する。異なる母数を指す人数を混同していないか（例: 部員数の文脈で合宿参加人数を使っていないか）。

## 報告形式

検証結果は以下の表形式で報告する。

| 項目 | 文書の記載 | 検証結果 | 判定 |
|---|---|---|---|
| （例）合宿初日の曜日 | 8月1日（土） | `datetime.date(2026,8,1).weekday()` → 5（土） | ✅ 一致 |

- エラーが見つかった場合は、判定を❌とし、**修正案とセットで**提示する（放置しない）。
- 検証対象の項目すべてで問題がなければ、末尾に「全項目確認済み」と明言する。
- 一つでも❌がある場合は、修正するまで配布不可であることをユーザーに明示する。
