# ラグビー部 メール・メッセージ下書き作成

ラグビー部関連の連絡文（メール・LINE）を作成するためのスキル。

## 手順

1. **まず `docs/rugby/基礎データ.md` を読む。**
   署名・取引先連絡先・銀行口座・合宿確定値など、連絡文に必要な情報はすべてこのファイルにまとまっている。書き始める前に必ず参照すること。

2. **ビジネスメール（学校事務・ミズノ・菅平ホテル・国際興業・保護者など）の書き方**
   - 署名は基礎データの「メール固定署名」を**一字一句そのまま**使用する。改変・省略・体裁変更は不可。
   - 書き出しは宛名「○○様」のあと、「いつもお世話になっております。駒澤大学高等学校ラグビー部顧問の占部です。」で始める。
   - 文体は丁寧・簡潔にまとめ、要件が一目で分かる構成にする（挨拶→用件→依頼事項・確認事項→結び→署名）。

3. **畠山先生（共同顧問）へのLINE返信の書き方**
   - 短文・口語敬体でまとめる（例:「承知しました」「〜します」）。
   - ビジネスメールのような堅い書き出しや署名は不要。
   - こちらのミスを指摘された場合は、言い訳せず素直に一言お詫びを入れる。

4. **Gmail MCPツール（`create_draft`）を使う場合の注意**
   - 添付ファイルは付けられない。添付が必要な連絡の場合は、下書き作成後に「添付ファイルは手動で添付してください」と案内する。
   - MCPツールが「requires approval」等のエラーを返した場合は再試行せず、本文をそのままチャットに出力し、ユーザーにコピペしてもらう。

5. **数値の扱い**
   - 金額・日程・人数などの数値は、必ず `docs/rugby/基礎データ.md` か会話中に提示された一次資料から取得する。記憶や推測で数値を書かない。
   - 基礎データに無い数値・未定事項（例: コーチ謝礼の合宿1人あたり配分、振込期限など）を書く必要がある場合は、断定せず「要確認」である旨を明記するか、ユーザーに確認する。

6. **下書き作成後の確認**
   - 下書きを作成したら、宛先・件名・本文を要約してユーザーに提示し、送信前に確認を得る。
