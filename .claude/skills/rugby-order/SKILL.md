# 用具・ウェア注文

## 0. 先に読むもの

作業を始める前に、必ず `docs/rugby/基礎データ.md` を読むこと。価格表・ミズノ緒方さんの連絡先は記憶で書かず、同ファイルの内容を正とする。

## 1. 年間の注文タイミング

- 4月下旬: 新入部員一式（採寸 → 見積 → 発注）
- 10〜12月: 冬用ウォーマー

## 2. 発注フロー

1. 採寸／フォーム回答を収集する
2. 見積を依頼する。この見積依頼の目的は金額確認だけでなく、毎年の品番・価格改定の有無を確認することにある。前年の価格表をそのまま流用しない。
3. 保護者向け案内を作成する（`rugby-doc` スキルを使用）
4. 発注する
5. 納品確認を行う

## 3. 支払いについて（重要）

- **2026年度からミズノが各家庭へ個別請求する方式に変更されている。部の口座への振込は不要。**
- 過去に案内書へ誤って振込案内を載せてしまい、訂正を出した経緯がある。案内書には振込先を一切書かないこと。案内文言を作成・レビューする際は、振込先の記載がないことを必ず確認する。

## 4. サイズ集計の規約

- 一次データはGoogleフォーム回答（タイムスタンプ付き）を正とする。
- 整列済みの注文表を扱う際、別品目のサイズ列を混同しないこと。過去にラグビーパンツのサイズ列とショルダーガードのサイズ列を取り違えた事故がある。品目名と列の対応を必ず確認してから集計する。
- 同一人物から複数回回答がある場合は、最新のタイムスタンプの回答を採用する。
- 未回答者は注文表・集計表で黄色網掛けにし「未確認」と明示する。放置せず本人確認を促すこと。
- 生徒名簿・注文内容（氏名・クラス・出席番号・品目・サイズ・数量）の正データは `scripts/create_handouts.py` の `STUDENTS`（および `PRICES` などの価格定義）とする。個別データが必要な場合は同ファイルを参照し、`docs/rugby/基礎データ.md` には転記しない。

## 5. 注文票の作成

- 注文票は「品名・サイズ・数量・生徒名」の一覧表と、「サイズ別集計表」をセットで作成する。
- Excel形式が希望される場合は openpyxl を使用する。

## 6. 注文メール

- ミズノへの発注メールなど業者宛のメールは `rugby-mail` スキルの規約（固定署名を含む）に従って作成する。
