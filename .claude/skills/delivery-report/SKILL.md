# 納品時の報告フォーマット

作業完了の報告は、必ず以下の6項目で構成する。初心者でもそのまま使える具体性で書く。

1. **変更したファイル** — パスと変更概要
2. **追加した機能** — 何ができるようになったか
3. **使い方** — 設置手順・実行手順（貼り付け場所、メニュー操作まで具体的に）
4. **テスト方法** — 動作確認の手順（必ずダミーデータを使う）
5. **注意点** — 制約・既知の限界・権限設定など
6. **次に改善できること** — 任意の改善提案

完了したタスク指示書があれば `tasks/doing/` から `tasks/done/` へ移動し、必要に応じて README.md と docs/ を更新する。
