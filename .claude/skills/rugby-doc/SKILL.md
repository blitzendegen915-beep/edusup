# rugby-doc: ラグビー部 正式Word文書作成

駒澤大学高等学校ラグビー部の保護者・生徒・学校向け正式文書（案内書・お知らせ・徴収文書など）をpython-docxで作成する。

## 手順

### 1. 基礎データを先に読む

`docs/rugby/基礎データ.md` を必ず読んでから作業を始める。学校名・顧問名・口座情報（振込用/入金用の区別）・確定済みの数値（部員数・合宿参加人数など）はここが正とする。記憶や推測で埋めない。

### 2. 雛形の有無を確認する

**同種文書の雛形（昨年版など）が存在する場合は必ずそれを使う。** ゼロから組むのは雛形が存在しない場合のみ。

雛形がある場合の方式（`scripts/create_camp_notice.py` が実例）:

- `shutil.copy2()` で雛形docxを出力先にコピーしてから開く（雛形自体は書き換えない）
- `python-docx` で段落・runを取得し、**runのテキストだけを書き換える**（既存の書式・レイアウトを保持するため、新規runを作る場合もフォント・サイズを既存runからコピーする）
- 1つの段落内の複数runを丸ごと空にしてから `para.runs[0].text = ...` で書き直す、または対象runを走査して部分置換する（`replace_in_para` / `set_all_runs_text` のようなヘルパー関数を用意すると安全）
- 新しくrunを追加する場合は必ずフォント設定をセットする（下記3を参照）

### 3. ゼロから作る場合の書式規約

雛形がない場合、`scripts/create_torokahi_notice.py` を参考に以下を守る。

- **用紙**: A4（`Cm(21.0)` × `Cm(29.7)`）。余白はCmで明示指定する。
- **フォント**: 游明朝で統一する。`run.font.name = '游明朝'` に加えて **`run._element.rPr.rFonts.set(qn('w:eastAsia'), '游明朝')` を必ず設定する**（これがないとWordが日本語部分に游明朝を適用しない）。`doc.styles['Normal']` にも同様に設定する。
- **レイアウト順**:
  1. 日付・差出人 — 右寄せ（`WD_ALIGN_PARAGRAPH.RIGHT`）
  2. 宛名 — 左寄せ
  3. 件名 — 中央揃え・太字・14pt程度
  4. 前文（挨拶文）
  5. 「記」 — 中央揃え・太字
  6. 詳細（表や箇条書き）
  7. 「以上」 — 右寄せ
- 表を使う場合は `table.style = 'Table Grid'`、列幅は `Cm()` で明示する。

### 4. 未確定・仮の値は黄色網掛けにする

日付未定・金額未定・対戦校未定など、ユーザー確定前の仮値は必ず黄色網掛け（`w:shd` の `fill=FFFF00`）で視覚的に区別する。`create_camp_notice.py` の `highlight_run()` をそのまま流用してよい。

```python
def highlight_run(run, color='FFFF00'):
    rPr = run._element.get_or_add_rPr()
    for old in rPr.findall(qn('w:shd')):
        rPr.remove(old)
    shd = OxmlElement('w:shd')
    shd.set(qn('w:val'), 'clear')
    shd.set(qn('w:color'), 'auto')
    shd.set(qn('w:fill'), color)
    rPr.append(shd)
```

ユーザーが値を確定させたら、該当runから `w:shd` を除去して網掛けを外す。網掛けを外し忘れたまま配布しない。

### 5. 生徒個人別文書（17名分など）の方式

生徒ごとにページを分けて配布する文書（用具購入案内など）は `scripts/create_handouts.py` の方式に従う。

- `STUDENTS` のようなリスト（出席番号・氏名・クラス・注文内容）をコードの先頭にまとめる
- 生徒ごとに `doc.add_page_break()` でページを分ける
- **金額（単価×数量、合計）は必ずコードで計算する。手計算した値をそのまま書き込まない。** `PRICES` のような単価辞書を用意し、`unit * qty` の合計をループで積み上げる。
- 生成後、期待値と突き合わせるログ（例: `create_handouts.py` 末尾の合計確認ループ）を出力し、目視でも確認できるようにする。

### 6. 出力・保存

- 生成したdocxは `output/` に保存する。
- 生成スクリプトは `scripts/` に保存し、再実行すれば同じ文書を再生成できる状態にする（使い捨てスクリプトにしない）。
- 完成したファイルはSendUserFileでユーザーに送る。

### 7. 配布前ファクトチェックを必ず行う

**保護者・外部向け文書は、配布前に必ず `rugby-check` スキルの手順でファクトチェックする。** この部では過去に曜日誤り（実際と異なる曜日を記載）やサイズ誤記（ショルダーガードのサイズ違い）を配布してしまった実績があるため、省略しない。

### 8. 完了後のコミット

作業が完了したら `git add` / `git commit` / `git push` を行う。ブランチ運用は本リポジトリの既存ルール（作業ブランチで作業しmainへは通常の手順でマージ）に従う。コミットメッセージは日本語で内容を簡潔に要約する。
