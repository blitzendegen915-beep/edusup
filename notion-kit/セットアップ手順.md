# Notion運用キット セットアップ手順（所要時間 目安10分）

対象: 駒澤大学高等学校ラグビー部 顧問（占部涼也）
前提: Notionアカウントは**顧問のみ**が持てば十分です。生徒・保護者はアカウント登録不要（「Web公開」URLを見るだけ）。

このキットで作るもの:
- 顧問専用の管理ページ「🏉ラグビー部ポータル」（非公開）
- 生徒・保護者に配る「📢」ページ 5枚（Web公開URLをLINE等で配布）
- Googleフォームでの食事・体調記録の収集（生徒はNotionに入らない）

同梱CSVファイル（このあとNotionに取り込みます。中身はダミーの見本行です。取り込んだ後、実データに書き換えてください）:
- `試合カレンダー.csv`
- `連絡板.csv`
- `食事記録.csv`
- `タスクボード.csv`

---

## ステップ0：事前準備（1分）

1. 顧問（占部涼也）のNotionアカウントでログインする。
2. 同梱の4つのCSVファイルを、自分のPC（デスクトップなど分かりやすい場所）に保存しておく。
   - ドラッグ&ドロップで使うので、Notionをブラウザ／デスクトップアプリで開いた状態で、CSVファイルの場所もすぐ開けるようにしておくとスムーズです。

---

## ステップ1：「🏉ラグビー部ポータル」ページを新規作成する（1分）

1. Notion左サイドバーの「＋ ページを追加」（または「Add a page」）をクリック。
2. ページタイトルに `🏉ラグビー部ポータル` と入力する（絵文字はタイトル入力欄の先頭で「:rugby:」と打つか、アイコン追加から選べます）。
3. このページは**共有しない＝非公開のまま**にする。左上の「共有」ボタンを押しても「Web公開」はオンにしないこと（このページには生徒個人の食事記録やタスクなど非公開情報を置くため）。
4. これが以後、顧問が日々更新に使う「司令塔」ページになります。

---

## ステップ2：4つのデータベースを作る

「🏉ラグビー部ポータル」ページを開いた状態で、以下(a)〜(d)を順番に行います。やり方はすべて共通パターン（CSVをドラッグ&ドロップ→データベース化→ビュー追加）です。

### 共通の取り込み手順
1. エクスプローラー（Finder／ファイル管理）から対象のCSVファイルを、Notionページ本文の中に**ドラッグ&ドロップ**する。
2. Notionが自動でファイルを認識し、「CSVをインポートしますか？」に近い確認、またはファイルカードが表示される。ファイルカードをクリックすると「テーブルとして表示」「データベースに変換」といった選択肢が出るので、**「データベースに変換（Convert to database）」**を選ぶ。
   - 表示されない場合：一度そのままアップロードし、ファイルカードの「…」メニューから「データベースに変換」を選ぶ。
3. 変換後、1行目（見出し行）がプロパティ（列）として認識されているか確認する。日付や選択肢の列は、列名をクリック→「プロパティを編集」で型が「日付」「セレクト」などになっているか確認し、違っていれば修正する。
4. データベースのタイトルを分かりやすい名前に変更する（例：「試合カレンダー」）。

### (a) 試合カレンダー
1. 上記の共通手順で `試合カレンダー.csv` を取り込み、データベース名を「📅試合カレンダー」にする。
2. データベース右上の「＋ ビューを追加」→「カレンダー」を選択。
3. 日付を表すプロパティ（例：「試合日」）をカレンダーの基準日付として指定する。
4. これで試合日をカレンダー形式でも一覧できるようになる。試合が決まったら、この表に1行追加するだけでよい（詳細は末尾「更新運用」参照）。

### (b) 連絡板
1. 同様の手順で `連絡板.csv` を取り込み、データベース名を「📣連絡板」にする。
2. 列に「対象」（全体／保護者／1年／2年／3年など）、「タイトル」、「本文」、「投稿日」があることを確認する。「対象」列がセレクト（複数選択可なら「マルチセレクト」）型になっているか確認する。
3. このデータベース自体は🏉ポータル配下の非公開DBのままでよい（直接公開しない）。ステップ3で、配布用ページからこのDBの「対象」でフィルタした表示（リンクドビュー）を埋め込みます。

### (c) 食事コンディション記録（顧問専用・絶対に公開しない）
1. 同様の手順で `食事記録.csv` を取り込み、データベース名を「🍚食事コンディション記録（非公開）」のように、非公開であると分かる名前にする。
2. 列に「氏名」「日付」「体重」「体調（5段階）」などの個人情報が含まれることを確認する。
3. **このデータベースは公開ページからは絶対にリンクしない。** 顧問がこのポータル内で直接開いて確認する専用データベース。
4. 実際の記録は主にGoogleフォーム経由で集まる（ステップ5参照）。このDBはその転記先・保管場所として使う。

### (d) 顧問タスクボード（非公開）
1. 同様の手順で `タスクボード.csv` を取り込み、データベース名を「✅顧問タスクボード」にする。
2. データベース右上の「＋ ビューを追加」→「ボード」を選択。
3. 「グループ化の基準」を「ステータス」列に設定する（未着手／進行中／完了 などの列があること）。
4. これで合宿準備・提出書類・発注などのタスクをカンバン形式で管理できる。このボードも公開しない。

## ステップ3：配布用ページ（📢）5枚を作る（3分）

生徒・保護者が見るのはこの5枚だけです。「🏉ラグビー部ポータル」ページの中（配下）に、以下5つのサブページを作ります。

1. ポータルページ内で「＋ ページを追加」を5回行い、それぞれ次のタイトルにする。
   - `📢全体連絡`
   - `📢保護者向け`
   - `📢1年`
   - `📢2年`
   - `📢3年`
2. 各ページの本文冒頭に、`ページ本文_配布用5ページ.md` に用意した文章をコピー&ペーストする。
3. 各ページに、連絡板データベースの**リンクドビュー**を埋め込む。
   - ページ本文で `/連携されたビュー`（または `/Linked view of database`）と入力し、「連絡板」データベースを選択。
   - 埋め込んだビューの「フィルター」を設定する。
     - 例：`📢保護者向け`ページ → 「連絡板」の「対象」列に「保護者」または「全体」を含む行のみ表示するフィルターを設定。
     - 同様に `📢1年` は「対象」に「1年」または「全体」を含む行、`📢2年`・`📢3年` も同じパターン。
     - `📢全体連絡` は「対象」に「全体」を含む行のみ、またはフィルターなしで全件表示でもよい（運用しやすい方でよい）。
   - ビューの並び順は「投稿日」の降順（新しい連絡が上）にしておくと見やすい。
4. 各ページのビューには「試合カレンダー」のリンクドビュー（カレンダー表示）も追加しておくと、生徒・保護者がそのページだけで日程と連絡の両方を確認できる。

> ポイント：連絡板DBは1つだけ。各📢ページには「そのDBを対象別にフィルタした窓」を置いているだけなので、連絡板DBに1行追加すれば、対象に合致する📢ページに自動で反映されます。

## ステップ4：Web公開する（2分）

**公開してよいのは📢の5ページのみ。「🏉ラグビー部ポータル」本体・連絡板DB・カレンダーDB・食事コンディション記録DB・タスクボードDBは公開しないこと。**

各📢ページで以下を行う：

1. ページ右上の「共有」をクリック。
2. 「Web公開」（Share to web / Publish）をオンにする。
3. 公開設定の中に「検索エンジンでの表示を許可」「Search engine indexing」のようなトグルがあれば**オフ**にする（Googleなどの検索結果に出さないため）。
4. 発行されたURL（`https://xxxx.notion.site/...`）をコピーする。
5. コピーしたURLをLINE公式アカウント・学年連絡網・保護者会LINEなどで、該当する対象者に配布する。
   - `📢全体連絡` → 部員・保護者全員に共有可
   - `📢保護者向け` → 保護者向けLINEグループ等
   - `📢1年`／`📢2年`／`📢3年` → 各学年の連絡網
6. URLはブックマーク推奨として案内する（毎回リンクを探させない）。

### 公開時の注意（必読）
- 公開するのは📢ページだけ。他のページ・データベースは絶対に公開しない。
- 検索エンジンインデックスは必ずオフにする。
- 📢ページ・連絡板DBには**個人名・個人成績・体重などの個人情報を書かない**（詳細は `運用ルール.md` を参照）。
- 生徒・保護者はNotionアカウントを作る必要はない。配布されたURLをブラウザで開くだけで閲覧できる。

## ステップ5：Googleフォーム連携（食事・体調記録）

生徒からの入力は、部で運用実績のあるGoogleフォームを使う。生徒をNotionに入れないことで、生徒同士・家庭同士が互いの記録を見られない設計にする。

1. Googleフォームで「食事・体調記録フォーム」を新規作成する。項目例：
   - 氏名（プルダウンで選択式にする。自由記述にすると表記ゆれが起きるため）
   - 体重（数値）
   - 朝食の内容（記述）
   - 昼食の内容（記述）
   - 夕食の内容（記述）
   - 補食の内容（記述）
   - 体調（5段階：1.とても悪い〜5.とても良い、のような選択式）
   - 記録日（日付）
2. フォームの回答は自動でGoogleスプレッドシートに集約されるよう設定する（フォーム編集画面の「回答」タブ→スプレッドシートのアイコン）。
3. フォームのURLは生徒（保護者経由でもよい）に個別に配布する。**この入力用URLは📢の公開ページには載せない**（回答フォームへの入力導線と、閲覧専用の📢ページは別物として管理する）。
4. 月1回、または顧問が気になる生徒がいたタイミングで、以下いずれかの運用を行う。
   - （a）回答スプレッドシートをCSVでエクスポートし、🏉ポータル内「🍚食事コンディション記録（非公開）」データベースにドラッグ&ドロップして取り込む（既存データに追記する形でよい）。
   - （b）全件取り込まず、体調評価が低い生徒・体重変化が気になる生徒だけを顧問が目視でNotionに転記する（軽量運用）。
5. どちらの方法でも、このデータベースは顧問のみが閲覧し、公開ページからは一切リンクしない。

---

## ステップ6：日々の更新運用（1分でできる）

- 試合が決まったら → 「📅試合カレンダー」データベースに1行追加するだけ。対象の📢ページ（カレンダービューを埋め込んでいる）に自動反映される。
- 連絡事項があったら → 「📣連絡板」データベースに1行追加（対象学年・タイトル・本文・投稿日を入力）するだけ。対象に一致する📢ページに自動反映される。
- 生徒個別の対応が必要な食事・体調記録が見つかったら → 「🍚食事コンディション記録（非公開）」で確認し、必要に応じて畠山監督にLINEで個別連絡する。

以上でセットアップは完了です。詳しい運用ルールは `運用ルール.md`、ポータル・配布ページに貼る本文は `ページ本文_ポータル.md` と `ページ本文_配布用5ページ.md` を参照してください。
