# 解答用紙作成スキル

## 原則

1. **教員が提供した最新テンプレートを常に基準にする。**
   現行: `materials/answersheet_comm2_template_new.docx`
   （「今後はこれを基にして」と指定済み。古いテンプレートは使わない）
2. 解答用紙（空欄）と模範解答（記入済み）は**同じスクリプトから生成**する。
   実装: `scripts/make_answersheet_comm2.py` の `build(model=False/True)`
3. 問題を変更したら、必ず解答用紙スクリプトの `ANSWERS` も同時に更新する。
   **過去の事故**: 大問6の模範解答が5問全部間違ったまま残っていた。

## ワークフロー

1. 問題用紙（確定版）から全設問・解答形式を抽出する
2. テンプレートの表構造を調査する（表番号・行列数・ラベル）
   ```python
   for ti, t in enumerate(doc.tables):
       print(ti, len(t.rows), 'x', len(t.columns), [c.text for c in t.rows[0].cells])
   ```
3. 設問数と枠数が合わない表は `exam-docx` スキルの表再構築パターンで直す
4. `ANSWERS` 辞書に模範解答を記入する。**各解答は問題用紙から独立に
   解き直して検証する**（前のスクリプトの答えをコピーしない —
   大問6全問誤答の原因はこれ）
5. `build(model=False)` → 解答用紙、`build(model=True)` → 模範解答
6. `exam-verify` スキルで3点セットを精査する

## 枠の設計

- **1枠1語**（教員指定）。「前後の単語を書く」形式は前用・後用の2枠。
- ラベル（設問番号）はテンプレートと問題用紙でズレていないか確認。
  過去例: テンプレートの `2A`/`2B` ラベルを新形式に合わせ `2`/`3` に書き換えた。
- 記述解答（和訳等）の枠は行の高さ・フォントサイズ(8pt程度)を確保する。
- 点数表示（/S10 /L10 /100 等）が満点と一致しているか確認する。

## 白文字ヒント

頭文字ヒント付き解答欄（`u` だけ黒、残り白文字）はテンプレート由来の
意図的な手法。**変更・削除しない**。ただしデータ配布時の注意点は
`exam-docx` スキル参照。
