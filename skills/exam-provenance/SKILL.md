# 出典管理スキル

## 背景（実際に起きた問題）

- 「大問1ってどうやって作った？」に答えられず、過去ログを掘り返して
  ようやく出典（Heartening II ワークブック）が判明した。
- 大問6の出典（動画でわかる英文法 例文51〜57）と大問1の出典を混同した。
- セッションが変わると添付ファイルやログは失われる。**リポジトリ内の
  ファイルだけが永続する。**

## ルール

1. **設問を1問でも作成・変更・差し替えたら、同じコミットで出典記録を更新する。**
2. 出典記録は `sources/<試験ID>.md` に置く（例: `sources/2026-comm2-1kimatsu.md`）。
3. 教材をスキャンしたPDF・画像を受け取ったら、`materials/` にコピーして
   コミットする（アップロード領域 `/root/.claude/uploads/` はセッション間で
   消えるため、参照した資料は必ずリポジトリに残す）。
4. **出典が確認できない文は試験に入れない。** 創作は禁止。
   ユーザーが自作した文はその旨を出典欄に明記する（例: `出典: 教員自作`）。

## 出典記録のフォーマット


## 差し替え時のルール

- どの問題をどれと差し替えたかを**ユーザーに明示**する（黙って変えない）。
- 差し替え候補は必ず出典教材の中から挙げる。候補には出典番号を付す。
- 変更履歴を出典記録の末尾に追記する:
  ```markdown
  ## 変更履歴
  - 2026-07-XX 大問1(2): crisis → universal（別解のため差し替え）
  - 2026-07-XX 大問1(6): rush → conduct（重複のため差し替え）
  ```
