# 定期考査 最終精査スキル

試験問題・解答用紙・模範解答の3点セットを配布前に精査する。
以下のチェックは**全て過去に実際に発生した問題**に基づく。省略禁止。

## 使い方

```
/exam-verify <問題用紙.docx> <解答用紙.docx> [模範解答.docx]
```

## 検査手順

### Step 0: docxは「段落＋表＋XML」の3層で読む（重要）

**過去の失敗**: 段落(paragraphs)だけを見て「大問4の語句リストが無い」と誤報告した。
リストは表(table)の中にあった。

- `python-docx` で `doc.paragraphs` と `doc.tables` の**両方**を必ず走査する
- それでも見つからない要素は `unzip` して `word/document.xml` を直接 grep する
  （テキストボックス `w:txbxContent`、ヘッダー/フッターも含む）
- 「無い」と報告する前に必ずXMLレベルで確認すること

### Step 1: 解答の一意性（別解チェック）

各問について「模範解答以外に正解となりうる答えが無いか」を検証する。

- **並び替え問題**: 全語句の順列で文法的に成立する別の並びが無いか。
  特に「不定詞句・副詞句を文頭に移動できないか」を確認する。
  - 実例: `What you need is to get enough sleep.` は
    `To get enough sleep is what you need.` という別解が成立した。
    → `is to get` を1チャンクにして解消した。
  - 対策の詳細は `skills/exam-unique-answer/SKILL.md` を参照
- **空所補充**: 同じ品詞・意味で他の語が入らないか。
  - 実例: `universal` は別解防止のため頭文字 `u` をヒント表示した。
  - 実例: 副詞 `once` は文中の複数箇所に挿入可能だったため語句を変更した。
- **語群選択**: 語群内に2箇所以上に入りうる語が無いか。冠詞(a/an)・単複・
  時制で一意に絞れているかを根拠として書き出す。

### Step 2: 出典の照合

- 全ての問題文・例文が実際の教材（教科書・ワークブック・副教材）に存在するか。
  **創作された文が混ざっていないか**を `sources/` の出典記録と照合する。
- 出典記録が無い場合は `skills/exam-provenance/SKILL.md` に従って作成する。
- 本文（読解パッセージ）が原文から**一切改変されていない**ことを確認する。
  - 実例: 「as shown in the pie charts below」を勝手に削除して問題になった。
    図表への言及・文の削除・語の置換はすべて禁止。改変が必要な場合は
    必ずユーザーに明示して承認を得る。

### Step 3: 構造整合性

- [ ] 配点: 各大問の配点×問数の合計が満点と一致（例: 筆記80+S10+L10=100）
- [ ] 小問番号が連番でズレていない
- [ ] 問題用紙の設問数 = 解答用紙の解答欄数 = 模範解答の解答数
- [ ] 指示文と実体の一致:
  - 「下の5つの語句」→ リストが実際に問題用紙に印字されているか（表も見る）
  - 「ア〜エから選べ」→ 選択肢が実際にア〜エの4つあるか
  - 「使わないものが2つある」→ 語群の数 − 空所の数 = 2 か
- [ ] 解答例(No.0)がある場合、例が本文と整合しているか
- [ ] 選択肢の正解番号が偏っていないか（全部①など）

### Step 4: 隠しデータ・体裁

- [ ] **白文字テキスト**: ヒント表示のため文字色を白(FFFFFF)にした残骸が
      ないか確認する。紙配布なら見えないが、**PDF/データ配布だと全選択
      コピーで答えが露見する**。実例: 解答欄の `u`(黒) + `niversal`(白)。
  ```python
  for p in doc.paragraphs:  # 表のセル内も同様に
      for r in p.runs:
          if r.font.color and str(r.font.color.rgb) == 'FFFFFF' and r.text.strip():
              print('白文字残骸:', repr(r.text))
  ```
- [ ] 赤字（訂正前の印）が残っていないか。run の color が FF0000 系を検出。
- [ ] 結合セル(gridSpan/vMerge)により同じテキストが複数セルに見える場合が
      あるので、誤検出に注意（cell.text が重複して見えるのは正常）。

### Step 5: 報告

- 「問題なし」項目と「要修正」項目を分けて表形式で報告する。
- 要修正項目には**根拠（該当箇所の実テキスト）**と修正案を必ず添える。
- 確信が持てない指摘は「要確認」として区別する。誤検出だった過去例:
  白文字ヒントを「解答が漏れている」と誤報告、表内リストを「無い」と誤報告。
